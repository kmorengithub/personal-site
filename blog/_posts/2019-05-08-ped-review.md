На этой неделе я сделал несколько важных вещей:
- Разработал форму отзыва научного руководителя о результатах
  педагогической практики, разместил ее [в
  интернете](https://konstantin-morenko.github.io/ped-practice-review/)
- Наконец настроил свой аккаунт на github на работу с ssh-ключем

На этих длинных выходных планирую:
- Закончить работу над темой для страницы педагогической практики
- Освоить основной ряд клавиш по раскладке dvorak
- Сделать fork репозитария с readtheorg темой

В fork я добавил функцию JS для того, чтобы хорошо отрисовывать
пункты, когда они имеют в своем составе несколько абзацев.  Я добавил
в заголовок org (с помощью `#+HTML_HEAD`):

    <script type="text/javascript">
	  $(document).ready(function() {
	    $('ol li:has(p)').css("margin-top", "24px");
		$('ul li:has(p)').css("margin-top", "24px");
	  })
	</script>

Но позже мне пришла в голову идея: а что если некоторые пункты не
будут иметь пояснений (в виде тегов `<p>`)?  Ведь в любом случае если
хотя бы один из пунктов будет с пояснением, то этот интервал нужно
будет ставить всем пунктам в пределах одного перечисления.  Но над
этим я еще не работал.

Еще я добавил hook в git перед выполнением коммита (файл
`.git/hooks/pre-commit`), чтобы не забывать пересобирать и проверять
файлы:

    #!/bin/sh

    # Check the rebuild files are up-to-date
    if ! $(make -q) ; then
      echo "Remake files with make"; exit 1
    fi

Сначала `make -q` проверяет, что файлы нужно пересобирать, и если дело
так, то выводит сообщение и прерывает коммит.
